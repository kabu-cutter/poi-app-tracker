# Changelog

## v0.8.3

- バックアップ安全策を追加
  - 一覧画面で最終JSONバックアップ日時・件数・対象バージョンを表示
  - バージョン更新後やJSONバックアップ未作成時に、JSONバックアップ推奨案内を表示
  - 案内からワンクリックでJSONバックアップを書き出せるように変更
  - バックアップ案内を「あとで」閉じられるように変更
  - 更新後の初回一覧表示時に、現在の `appProgressItems` を内部控え `appProgressItemsAutoBackup` として保存
  - JSON復元前・案件削除前にも内部控えを保存
  - 必要時に「内部控えから復元」できるボタンを追加
  - ポップアップにもバックアップ推奨の小さな案内を追加
- `manifest.json` のバージョンを `0.8.3` に更新
- `storage.js` は変更せず、保存キー `appProgressItems`、保存形式、権限、外部送信なしの方針を維持
- 開発版のChrome自動生成アイコン表示を維持するため、`manifest.json` に `icons` / `action.default_icon` は追加しない

## v0.8.2

- 案件登録時のアプリ名候補ロジックを改善
  - ページタイトル由来候補と元ページURL由来候補を分けて評価
  - URLのパス・クエリパラメータからアプリ名候補を抽出
  - URL末尾のランダムIDらしい文字列を候補から除外
  - `iOS` / `Android` などの表記は候補内では保持しつつ、単独のプラットフォーム名だけをアプリ名候補にしないよう調整
  - タイトル候補とURL候補が食い違う場合は自動上書きせず、確認用候補として表示
  - 候補表示の「候補を使う」「Discard」を維持
- 一覧検索にプラットフォーム別表示オプションを追加
  - iOS / Android / iOS + Android / 未判定で絞り込み可能
  - 保存済みデータ形式は変えず、アプリ名・案件タイトル・URLなどから自動判定
  - 一覧カードに判定されたプラットフォーム表示を追加
- `manifest.json` のバージョンを `0.8.2` に更新
- 開発版のChrome自動生成アイコン表示を維持するため、`manifest.json` に `icons` / `action.default_icon` は追加しない
- 保存キー `appProgressItems`、保存形式、権限、外部送信なしの方針を維持

## v0.8.1

- 案件登録画面の入力補助を改善
  - アプリ名と案件タイトルを分けやすい説明を追加
  - サイト名候補をURLから推定
  - アプリ名候補をページタイトルと元ページURLから推定
  - 同じURLの既存案件がある場合に注意表示
  - 同じアプリ名の既存案件がある場合に注意表示
  - 候補表示に「Discard」を追加し、不要な候補を非表示にできるように変更
- 注意表示は保存をブロックしない方式を維持
- `storage.js` は変更せず、保存キー `appProgressItems` を維持

## v0.6.0

- 編集画面に「進行ログ」を追加
  - 日付つきログを複数保存可能
  - ログ追加・削除に対応
- 一覧画面に「最新ログ」を表示
- 検索対象に進行ログを追加
- CSV出力に「最新ログ」「進行ログ」を追加
- JSONバックアップ形式の内部バージョンを更新
- 既存データは維持し、ログ未入力の案件は空配列として扱う

## v0.5.0

- GitHub用のレポ構成に整理
- レポ名を `poi-app-tracker` に想定
- Chromeウェブストア表示名は `アプリ進行中` のまま維持
- アイコン指定を削除し、Chrome自動生成の「ア」アイコンに戻す
- 期限表示を強化
  - 今日まで
  - 明日まで
  - あと○日
  - 期限切れ○日
- 一覧の初期並び順を「期限が近い順」に変更
- 並び替えを追加
  - 期限が近い順
  - 更新が新しい順
  - 作成が新しい順
  - サイト名順
- ステータスに「承認待ち」「撤退」を追加
- 「現在の進行度」「問い合わせ番号」項目を追加
- バックアップJSON出力を目立たせる表示に変更
- CSV出力項目に進行度・問い合わせ番号を追加
