## 2. 重要な開発方針

### 既存データ保護

- 保存キー `appProgressItems` は維持する
- データ形式を変更する場合は後方互換を考える
- 既存ユーザーのデータを壊さない
- 破壊的変更を入れる場合は、必ずバックアップ導線を強化する
- v0.8.1 / v0.8.2 / v0.8.3 / v0.8.4 では `storage.js` を変更しない
- v0.8.1 / v0.8.2 / v0.8.3 / v0.8.4 では保存形式を変更しない
- v0.8.3 では、本体データ `appProgressItems` とは別に内部控え `appProgressItemsAutoBackup` とバックアップ情報 `appProgressBackupMeta` を追加する

### 権限

- 権限は最小限を維持する
- 現在の権限:
  - `activeTab`
  - `storage`
  - `scripting`
- `scripting` は、ユーザーが「この案件を記録する」を押した現在タブだけを対象に、ページ内の案件名・対応OSらしい情報を登録補助へ使うために使用する
- `host_permissions` は当面追加しない
- ページ内から取得した情報は登録画面の入力補助にのみ使用し、外部へ送信しない
- 外部通信を入れる場合は、目的・送信内容・プライバシー説明を再検討する
- v0.8.1 / v0.8.2 / v0.8.3 では権限追加なし
- v0.8.4 で `scripting` を追加し、ストア掲載情報・プライバシー説明・権限理由へ用途を明記する

### アイコン

- 通常の開発版では、できるだけChrome自動生成の「ア」表示を維持する
- `manifest.json` には `icons` / `action.default_icon` を入れない
- ストア管理画面側で提出に必要なアイコン画像を扱う
- アイコンファイル自体はリポジトリに残してよいが、開発版の表示を変える必要がない限り manifest から参照しない
- 今後アイコンを変える場合は、ストア版・GitHub版・開発版で表示差が出ないように確認する

### iOSについて

- iOSネイティブアプリ開発は行わない
- Xcode / SwiftUI / Share Extension / App Store向けiOSアプリはスコープ外
- スマホ対応を考える場合は、まず以下を検討する:
  - モバイルWeb版
  - PWA
  - Chrome拡張版とのJSON互換
  - スマホで入力しやすいフォーム

## 5. v0.8.3 作業版: バックアップ安全策版

### 目的

Chromeウェブストア更新やパッケージ入れ替えに備えて、ユーザーがJSONバックアップを取り忘れにくくする。  
通常の同一拡張IDでの更新では `chrome.storage.local` のデータが残る想定だが、削除・再インストール・プロファイル変更・ストレージ削除・実装ミスに備えて安全策を増やす。

### 今回反映した変更

- `manifest.json` の `version` を `0.8.3` に更新
- 一覧画面に最終JSONバックアップ日時・件数・対象バージョンを表示
- バージョン更新後やJSONバックアップ未作成時に、バックアップ推奨案内を表示
- バックアップ推奨案内からJSONバックアップを書き出せるようにする
- 「あとで」でバックアップ案内を一時的に閉じられるようにする
- 更新後の初回一覧表示時に、現在の案件データを内部控えとして保存
- JSON復元前・案件削除前にも内部控えを保存
- 必要時に「内部控えから復元」できる導線を追加
- 内部控えは同じ拡張機能内の簡易バックアップであり、拡張機能を削除すると一緒に消える場合があることを明記
- 削除・入れ直し・別PC移行の前にはJSONバックアップを書き出す必要があることを明記
- ポップアップにバックアップ推奨の小さな案内を追加

  - 外部ファイルではなく、同じ `chrome.storage.local` 内の控え
  - 拡張機能を削除すると、この内部控えも消える場合がある
- `appProgressBackupMeta`

## 10. 将来プラン: スマホ対応

### 現在の扱い

- スマホ対応はプロジェクトの将来プランに含める
- 現時点では実装へ着手せず、Chrome拡張版の安定化と使いやすさ改善を優先する
- 短期TODOや次回リリースの必須範囲には含めない
- iOSネイティブアプリ、Xcode、SwiftUI、Share Extension、App Store向けアプリ開発は当面行わない

### 将来検討する方向

1. Chrome拡張版のJSONバックアップを、スマホ側でも安全に読み書きできる形式として維持する
2. モバイルWeb版またはPWAの必要性を検討する
3. スマホで確認・更新しやすい簡易入力フォームを検討する
4. PC版とのデータ受け渡しは、まずJSON互換と手動バックアップを基本にする
5. 自動同期、共有機能、ページ情報の自動取得は必要性が明確になってから検討する

### 再検討するタイミング

- Chrome拡張版の登録・一覧・バックアップ機能が安定した後
- スマホだけで進行度やログを更新したい場面が増えたとき
- JSONの上書き・追加インポート・競合回避方針が整理できたとき
